\subsection{Common Test}\label{sec:commontest}

Why did I chose Common Test?

Well, primarily because I love Erlang and Common Test is well developed and 
full featured black box testing framework.  It can produce excellent documentation
alongside the test results and erlang can integrate well with just about any
third party testing framework.

Additionally, functional programming is awesome because of it's ease of 
extensibility.  In contrast to OOP there is only one pattern to learn and use;
functions! 

The only downside to CommonTest is that it might require a Domain Specific Language (DSL)
to make test creation and validation easier.  But, since this project is all about
showing off, that downside because an upside!
