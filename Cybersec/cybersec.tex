\section{Introduction}

This document will serve to provide evidence for an Agile Software Development
methodology.  We will take the idea of a Cybersecurity Organization providing
Business Plan and will use technologies to generate proofs, tools, help, etc to
implement the businesses plan.

It is intended as a ``living'' document, therefore it will be written and developed
in \LaTeX{}.  This will allow us to break the project into pieces, allow us to reference
those pieces in later work, provide good formatting capabilities, and perhaps
most importantly; it will allow us to integrate code and processes directly into 
the document.

While the document will not be substitute for other technologies like project 
management software, CI/CD pipelines, diagraming tools, and software development 
IDEs it will provice a centralized location from where we can aggregate everything.
